Status: ✓ BUG FIXED & VERIFIED

## Bug Description

`retrieveSimilarStrategies()` returned 0 results even for exact matches.
Root cause: Query embeddings used a different vocabulary than stored embeddings,
causing low similarity scores (0.707 for exact match, below 0.75 threshold).

Example: Storing "collect oak wood" then querying "collect oak wood" returned
similarity 0.707 because query embedding [0.577, 0.577, 0.577, 0, 0, 0]
had different dimensions than stored embedding [0.408, 0.408, 0.408, 0.408, 0.408, 0.408].

## Fix Applied

Removed `_embeddingCache` entirely. Changed to store `embedding_text` (raw string)
in KnowledgeGraph instead of pre-computed `embedding` arrays.

`retrieveSimilarStrategies()` now:
1. Calls `_rebuildVocabulary()` to rebuild vocab from all stored strategies
2. Generates query embedding using current vocabulary
3. Re-embeds each stored strategy from its `embedding_text` at query time
This ensures query and stored embeddings always use identical vocabulary and IDF.
