## Design Notes
- Simple JavaScript class pattern (no complex DSL)
- Each skill execute() is async and returns { success, outcome, error }
- Parameters schema uses simple object: { param1: 'type', param2: 'type' }
- Skills use bot and vision from constructor injection via context
- Skills are stateless (each execution is independent)
- Map-based storage for O(1) lookup
- Auto-registration in constructor
