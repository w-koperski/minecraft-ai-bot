KNOWN ISSUE: StateManager Lock Bug
====================================
The StateManager.withLock() method uses the data file path (e.g., state.json) 
as the lock file path instead of appending '.lock'. This causes EEXIST errors 
when trying to read after write within the same process.

Impact:
- /api/drives, /api/goals, /api/metrics return errors when state.json was 
  recently written by the same process
- Direct file reads work fine
- This is a pre-existing bug, not introduced by phase-2-enhancements

Mitigation: Dashboard uses file polling (2s interval) which reads the file 
directly, not through StateManager, so real-time updates work correctly.
